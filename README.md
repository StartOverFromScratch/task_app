## 1. プロジェクト概要

### 1.1 目的

定期・不定期タスクおよび大規模タスクの分割管理を行い、

* 実行漏れを防ぐ
* 「やったかどうか不明」をなくす
* 終わらないタスクを構造的に処理する
* 探索が広がりすぎることを制御する

ことを目的とする。

---

### 1.2 対象ユーザー

* 個人利用（単一ユーザー）

---

### 1.3 背景コンテキスト（重要）

本システムは、以下の思考特性を前提に設計されている。

#### ユーザーの思考特性

* 調査・探索が得意で楽しい（A 60%）
* 思考が分岐しやすい（C 40%）
* 不安型ではない（Bは少）
* 完璧主義ではないが「ゴールが曖昧だと終われない」
* 構造がシンプルになった瞬間に決断できる
* 可逆な決定であっても不可逆のように感じ、探索が長引く傾向がある

#### 設計思想

* 「最適解探索」ではなく「構造収束」で止める
* 可逆なら仮決定で良いという思想を明示的に組み込む
* 調査→決定→実行を分離する
* 終了条件（Definition of Done）を必須化する
* 探索の終了条件（件数制限）を明示する

## 2. スコープ

### IN

* タスク登録
* 成果物ベース分割（親子構造）
* 完了条件（1行DoD）
* 任意チェックリスト
* 決定基準（decision_criteria）
* 可逆判定（reversible）
* 探索件数上限（exploration_limit）
* 完了ログ保持
* 繰り越し候補管理（手動確定）
* 放置検知（最終更新日ベース）
* 要再定義フロー
* CaptureBox（探索分岐逃し箱）

### OUT（v0.2では作らない）

* チーム共有
* 外部連携
* 通知機能
* 高度な分析
* 認証の高度化

## 5. データモデル

### tasks

* id
* title
* task_type (research / decision / execution)

* priority (must / should)

## 6. 状態遷移

```
未着手 → 進行中 → 完了
　　　↘
　　　 要再定義 → 進行中
　　　↘
　　　 保留 → 未着手

期限超過 → 繰り越し候補 →（手動確定）→ 未着手
```

---

## 7. 放置検知ルール

基準：last_updated_at

* Must：7日
* Should：21日

---

## 8. 繰り越し仕様

期限超過タスクは carryover_candidate へ。

操作ボタンは固定：

* 今日やる
* +2日
* +7日
* 要再定義

---

## 9. 決定設計（探索収束プロトコル）

調査タスクには以下を必須：

* exploration_limit（例：3）
* reversible確認
* decision_criteria（合格基準）

終了条件：

* 選択肢が3つ以内
* 構造が単純化された
* 可逆であると確認できた

## 11. UI/UX 原則（v0.2追加）

### 11.1 タスク一覧のデフォルト挙動

| 項目 | 仕様 |
| ---- | ---- |
| デフォルトステータスフィルタ | `todo` / `doing` のみ表示（完了・保留・要再定義は非表示） |
| デフォルトソート | `due_date` 昇順（期限の近い順）。due_dateがnullのタスクは末尾に表示 |
| フィルタ切り替え | UIで任意変更可能（全ステータス表示も可） |

### 11.2 子タスクの一覧表示

| 項目 | 仕様 |
| ---- | ---- |
| 表示対象 | 親タスク・子タスクをデフォルトで両方表示する |
| 識別方法 | `TaskCard.vue` に「子タスク」バッジを表示（`parent_id != null` で判定） |
| 切り出し元の識別 | `origin_checklist_item_id` があれば「チェックリストから切り出し」バッジを追加表示 |

### 11.3 PCでの2カラムレイアウト

| 項目 | 仕様 |
| ---- | ---- |
| 適用ブレークポイント | `lg`（1024px以上）で2カラム表示 |
| 左カラム | タスク一覧（固定幅 360px） |
| 右カラム | 選択タスクの詳細（残り幅） |
| 未選択時の右カラム | プレースホルダー表示（「タスクを選択してください」） |
| モバイル | 従来通りページ遷移（`/tasks/[id]`） |

### 11.4 タスク詳細画面のタスク名称表示

タスク詳細画面（`/tasks/[id]`）のページタイトル部分に `title` を明示的に表示する。2カラム時は右カラムのヘッダーとしても機能する。
